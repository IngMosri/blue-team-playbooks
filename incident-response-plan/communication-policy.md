# Security Communication Policy

## Principle
End users must not be informed about specific monitoring capabilities. Communication should be abstract and privacy-aligned.

## Approved Language
- "Automated security alerts"
- "System detected anomalous activity"
- "Endpoint health validation"

## Prohibited Language
- "We saw you browsing..."
- "Your logs show..."
- "We monitor your traffic"

## Escalation
User communication must be approved by SOC L2 or Security Management.
